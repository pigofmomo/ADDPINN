# ADD-PINN

Official research code for **ADD-PINN: Adaptive Domain Decomposition based Physics-Informed Neural Networks via Spatial Clustering**. The paper has been accepted by **Communications in Computational Physics (CICP)** and is awaiting publication.

ADD-PINN extends PINNs with adaptive spatial domain decomposition. This repository includes global PINN, XPINN-style decomposition, and ADD-PINN variants with spatial clustering, banded interfaces, balanced initialization, gradient-analysis-based synchronization, and optional subdomain adjustment.

## Key features

- Adaptive irregular subdomains generated by spatial clustering.
- Banded interface loss for stronger coupling between subnetworks.
- Graph-optimization-based initialization and subdomain balancing.
- Optional dynamic subdomain adjustment during training.
- Benchmark scripts for function approximation, Poisson-Boltzmann, heat equation, lid-driven cavity, and complex domains.

## Installation

Install dependencies from the repository root. A virtual environment is optional.

```bash
pip install -r requirements.txt
```

`requirements.txt` pins DeepXDE to the 1.11 series. If you need a CUDA-specific PyTorch build, install `torch` from the official PyTorch instructions first, then run the command above.

## Quick start

Run examples from the repository root with module commands so package imports resolve correctly.

### 2-D function approximation

python -m pinn_add.function2d.function2d

## Basic usage

1. Define a DeepXDE geometry and build a `pinn_add.PointSets2D` point set.
2. Define either a target function with the `function` key or a PDE residual with the `pde` key.
3. Provide one network for a global PINN or multiple networks for decomposed models.
4. Configure decomposition options such as `interface_type="line"`, `interface_type="band"`, `initial_subdomain_balance=True`, and `adjust_subdomain=True`.
5. Train with `PINNADD.train()`, then save or evaluate the result.

```python
import deepxde as dde
import pinn_add

spatial_domain = dde.geometry.Rectangle([-1, -1], [1, 1])
pointset = pinn_add.PointSets2D(spatial_domain, divide_config=None)

nets = [
    [2, 50, 50, 50, 50, 1],
    [2, 50, 50, 50, 50, 1],
    [2, 50, 50, 50, 50, 1],
]

data = {
    "function": target_function,
    "pointset": pointset,
    "num_domain": 10000,
    "n_clusters": 500,
    "interface_type": "band",
    "initial_subdomain_balance": True,
    "adjust_subdomain": True,
}

model = pinn_add.PINNADD(nets, data, save_dir="./pinnadd_result/0/")
result = model.train(iterations=20000, print_every=100)
model.save("./pinnadd_result/0/", result)
```

## Configuration

Global defaults are in `pinn_add/config_pinnadd.py`. Common settings include random seed control, learning-rate decay, pretraining iterations, adaptive subdomain update schedule, and graph-optimization parameters.

Example scripts override these values near the top of each experiment file.

## Outputs

Training scripts save results under experiment-specific folders such as:

- `pinn_result/<seed>/`
- `xpinn_result/<seed>/`
- `pinnadd_result/<seed>/`
- `pinnadd_result2/<seed>/`
- `pinnadd_result3/<seed>/`

Outputs may include model states, metrics, figures, and decomposition artifacts.
